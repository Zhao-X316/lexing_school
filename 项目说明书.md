## 1. 项目概述

- **项目名称**：数字化教学管理与展示系统
- **项目目标**：为机构搭建一套集「品牌展示、内容管理、智能教案生成、家校数据互通」为一体的私有化综合 Web 系统。通过数字化的四级标签匹配和动态模板技术，将教案编写工作自动化；通过可视化的成长图表实现学生能力长期追踪，提升教学效率与家校透明度。
- **技术栈**：
  - ***前端展示（官网）**：官网已转换为 Vue 架构，与管理端/家校端统一在 Vue 项目内。*
  - **前端业务（管理端/家校端）**：Vue.js，组件化管理；ECharts 实现雷达图与折线图。
  - **后端**：Node.js（配合 exceljs、docxtemplater 等 Excel/Word 处理库）。
  - **数据库**：MySQL（多表关联、权限划分、历史数据存储）。
  - **部署**：基于 Zspace NAS 的 Docker 容器化部署（数据库 + 后端 + 前端），配合内网穿透实现公网访问，数据私有化。
- **主要用户/场景**：机构管理员、教师、家长；教学管理、教案生成、成长追踪、家校互通。

## 2. 角色与权限体系 (RBAC)

系统采用严格的四级权限隔离机制，开发与修改须严格遵循下列权限边界。*现行角色定义与说明以 **docs/权限说明.md §1** 为准；依据该说明拆解的前后端待办规划见 **§12**。*

| 角色 | 权限范围 | 说明 |
|------|----------|------|
| **超级管理员 (Super Admin)** | 系统底层 | 负责 NAS 存储路径配置、数据库备份管理、全局系统参数设置及分配「管理员」账号；不直接操作业务数据。 |
| **管理员 (Admin)** | 业务总控 | 教师/学生/家长账号、给教师和家长号创建连接、教案库、模板、首页配置、上传/审核活动宣传。 |

| **家长 (Parent)** | 只读、自家孩子 | 一名家长可绑定多名学生；登录后进入专属看板，查看成长图表、历史教案、课时进度。 |

## 3. 功能模块

### 3.1 内容管理系统 (CMS) 模块

- **功能描述**：
  - 集成富文本编辑器，提供类似微信公众号文章的编辑体验，支持图文混排、视频插入，用于一键生成和发布活动宣传页。
  - 首页元素管理：通过表单接口允许管理员替换首页中的指定图片和视频链接；*轮播图与视频链接存入「首页配置表」见 §6.2（8）。*
- **验收标准**：管理员可登录后台完成活动页编辑与首页素材替换，前端展示与配置一致。

**首页动态化实施约束**（依据 `docs/首页动态化项目规划.md`）：

- **不改变设计布局**：首页动态化仅替换**内容**（文案、图片、视频链接、联系方式等），**不改变**官网首页的版式、区块顺序、样式（CSS）、视觉结构与交互逻辑。各区块的 HTML 结构、类名、布局方式保持不变。
- **实施方式**：官网 `onMounted` 时请求 `GET /api/homepage/config` 拉取配置，有则用配置渲染，无则使用与当前硬编码一致的默认值兜底；接口失败时静默降级，不白屏。
- **配置键与区块对应**：`hero`（首屏）、`philosophy`（核心理念）、`facilities`（设施展示）、`why_choose_us`（为什么选择我们）、`contact`（联系方式/页脚），与管理后台「首页配置」各 Tab 一一对应。

**首页图片/视频替换方案与优缺点（供实现选型）**：

| 方案 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **A. 数据库配置表** | 建 `homepage_config` 表，key-value 存轮播图 URL、视频 URL 等 | 与现有权限/CMS 一致、可审计、易扩展多图多视频 | 多一次 DB 查询（可缓存） |
| **B. 配置文件/JSON** | 后端或静态目录放 `config.json`，接口读文件写文件 | 无 DB 依赖、部署简单 | 多实例需共享存储、权限与备份需自行保障 |
| **C. 复用 articles 表** | 用文章类型区分「首页轮播」「首页视频」 | 不增表、富文本可复用 | 语义混杂、首页结构化字段（如多图顺序）不便 |

*本项目采用方案 A（数据库配置表），已增加 `homepage_config` 表，便于权限统一与后续扩展。*

### 3.2 智能教案生成与动态模板模块

**功能描述**：
  - **四级标签级联检索**：界面支持「7 大类 → 数个小类 → 3 个阶段 → 长短期目标」的折叠展开与勾选。
  - **关联长期目标**：勾选此选项后，勾选长期目标会自动勾选其关联的短期目标（按 `target_number` 前缀匹配，如 1-M 对应 1-a、1-b）；取消长期目标时，关联短期目标同步取消。
  - **动态模板引擎**：支持管理员后台上传 Excel/Word 模板；系统保存原始模板文件，并统一生成标准化模板（`.xlsx` / `.docx`）用于教案生成。未选模板时默认使用最近启用的 Excel 模板。
  - **模板在线预览**：后台模板管理支持在线预览模板的 **PDF 版本**；不直接依赖浏览器原生预览 Word/Excel。
  - **实时渲染与导出**：教师勾选标签后前端实时展示网页版教案预览；点击生成后，后端将所选目标数据填入模板占位符，生成排版一致的 `.xlsx` 或 `.docx` 供下载。
  - **下载命名**：导出文件名格式为 **{学生姓名}-{老师姓名}-{年月}.xlsx**（如：张三-李老师-2026年3月.xlsx）。
- **验收标准**：教师可完成标签勾选、预览、一键导出与下载，导出文件排版与上传模板一致。

**占位符设计规范**（制作 Excel/Word 模板及引擎实现须严格遵循）：

- **基本信息**：`{{studentName}}` 学生姓名；`{{assessmentDate}}` 评估日期；`{{teacherName}}` 负责教师。
- **循环列表**（多目标）：Word 中使用 `{#targets}` … `{/targets}` 包裹重复区域，区域内可用 `{{number}}`、`{{majorCategory}}`、`{{stage}}`、`{{content}}` 等；引擎按勾选目标数量循环输出行或段落。
- 模板中不得使用文档未约定的占位符；引擎仅替换上述约定占位符，保持其余排版不变。

**IEP Excel 模板约定**（智力康复个别化教育计划）：

- 表头固定第 4 行，数据从第 6 行起；第 1/2/3 列分别为康复领域、长期目标、短期目标。
- 短期目标分行插入（每个短期目标占一行）；领域与长期目标列纵向合并。
- `major_category` 支持短码（PSC）与完整标签（个人自理(PSC)）两种存储格式，引擎自动归一化匹配。

### 3.3 学生档案与数据可视化模块

- **功能描述**：
  - **纵向追踪（折线图）**：提取学生 1 月、3 月、5 月等历史评估得分或关键指标，绘制成长折线图。
  - **横向剖析（雷达图）**：基于单次评估中 7 大类的勾选情况，生成能力剖面雷达图。
- **验收标准**：教师/家长端可查看指定学生的折线图与雷达图，数据与评估记录一致。

### 3.4 课务管理模块

- **功能描述**：记录学生总课时、已消课时、剩余课时。课时数据自动联动并同步至家长端看板。

**课时扣减方案**（已确认）：

| 操作 | 是否扣课时 | 说明 |
|------|------------|------|
| 生成教案（下载） | **否** | 教案生成仅落库并导出文件，不扣减课时 |
| 上传打卡记录表 | **是** | 管理员/教师上传学生打卡记录表，系统解析并统计上课次数，按次扣减 `used_hours`、`remaining_hours` |

*系统内不做打卡录入功能。* 上传打卡报表时，系统会**记录每次上课的具体时间**，供家长端查看。

- **验收标准**：课时变更规则明确、数据一致，家长端展示的剩余课时与后台一致；家长可查看上课时间记录。

### 4.1 第二阶段：技术栈与实施路径（权限中枢与内容管理）

**技术栈约定**（须严格采用，不得随意替换）：

- **后端**：Express.js 或 Koa2；鉴权 **jsonwebtoken (JWT)**；数据库驱动 **mysql2**。
- **前端**：**Vue 3** (Composition API)；**Vue Router** + **Pinia**；UI 组件库 **Element Plus** 或 **Ant Design Vue**；富文本 **WangEditor 5**。

**六步实施顺序**：

1. **技术栈初始化**：后端 Express/Koa + mysql2 + JWT；前端 Vue 3 + Router + Pinia + UI 库 + WangEditor。
2. **核心鉴权与权限中枢（后端）**：实现 `/api/auth/login` 与 JWT 签发；编写四级权限中间件（解析 Token 的 role，超管独享系统级维护，管理员不可执行数据库备份/超管账号修改等）。
3. **管理端框架与动态路由（前端）**：后台骨架（左侧导航、顶部用户、右侧内容）；Token 存 localStorage 持久化；根据 role 动态生成菜单（超管全量/管理员隐藏系统维护/教师仅业务菜单/家长仅查看）；403 越权拦截。
4. **账号管理 API 落地**：`/api/users/list`、`/api/users/create`，严格挂载权限中间件；前端表格与表单对接。
5. **内容管理 (CMS) 与首页动态化**：WangEditor 嵌入与 `/api/articles/publish`；首页配置接口 `/api/homepage/update`；静态官网迁移至 Vue，onMounted 请求最新配置实现动态渲染。
6. **联调与越权测试**：全链路登录→Token→鉴权→数据库；管理员访问超管接口、教师访问管理接口的拦截验证；交付可运行后端 + 前端 + 动态化首页。

**阶段交付物**：可运行的 Node.js 后端（含上述 API 与 JWT 鉴权）；Vue.js 后台管理系统（登录页 + 动态菜单骨架）；初步动态化的官网首页（可由后台修改部分图文）。

### 4.2 第三阶段：技术栈与实施路径（核心业务攻坚）

**技术栈约定**：

- **前端**：级联选择器 (Cascader) / 树形控件 (Tree)（Element Plus 或 Ant Design Vue）；Axios `responseType: 'blob'` 接收文件流并触发下载。
- **后端**：**multer** 接收模板与标签 Excel 上传；**docxtemplater** + **pizzip** 处理 Word；**exceljs** 处理 Excel；（可选）**xlsx** 或 exceljs 解析标签库 Excel 批量导入。

**六步实施顺序**：

1. **依赖与存储**：安装 multer、docxtemplater、pizzip、exceljs；配置 NAS/本地模板存储路径。
2. **四级标签库（Admin 端）**：批量导入 API（解析 Excel 写入 `teaching_targets`）；后台 Tree 展示与增删改查。
3. **动态模板管理（Admin 端）**：模板上传接口（multer），路径与元信息写入 `templates` 表。
4. **教师端评估工作台**：级联/树形勾选四级目标；右侧“已选清单”实时预览。
5. **核心渲染引擎（后端）**：接收 student_id + target_id 数组；组装 JSON（学生信息 + 目标内容）；docxtemplater / exceljs 替换模板占位符，输出 Buffer。
6. **落库与下载闭环**：生成成功后写入 `assessments` + `assessment_details`；响应二进制流，前端 blob + `URL.createObjectURL` + 规范命名下载。下载文件名格式为 **{学生姓名}-{老师姓名}-{年月}.xlsx**（如：张三-李老师-2026年3月.xlsx）；后端通过 `Content-Disposition` 返回，CORS 需暴露 `Content-Disposition` 供前端读取。

**阶段交付物**：管理员端具备多级标签树展示与管理、教案模板上传与配置；教师端具备级联勾选与实时已选预览；稳定可用的 Node.js 模板渲染接口，完成 JSON→.xlsx/.docx 转化并触发浏览器下载。

### 4.3 第四阶段：技术栈与实施路径（数据可视化与家校端）

**业务目标**：激活历史数据、实现家校闭环。将评分记录转化为能力成长轨迹（折线图）与多维度能力剖面（雷达图）；为家长提供只读专属端；完善教案「软删除/作废」机制，保证数据可追溯。

**技术栈约定**：

- **数据可视化**：**ECharts** 或 Vue 3 封装 **vue-echarts**，用于雷达图与折线图动态渲染。
- **时间处理**：**Day.js**，用于评估日期格式化（如“2026年3月”）及折线图横坐标。
- **前端状态**：继续使用 Vue Router + Pinia 实现家长端权限隔离。

**三大模块实施路径**：

1. **ECharts 数据可视化引擎（学生个人页）**
   - **雷达图**：后端从 `assessment_details` 聚合单次评估同一「大类」得分（如 7 大类加权平均），返回规范 JSON；前端 ECharts 渲染 7 维度能力剖面。
   - **折线图**：后端按时间顺序拉取该学生所有 `status = 'active'` 的评估记录（1 月、3 月、5 月等），返回历次综合得分或分大类得分；前端折线图 + 交互（悬浮显示得分与日期）。
2. **家长端专属登录与看板**
   - 家长角色 (`parent`) 专属路由与视图：*一名家长可绑定多名学生，通过列表切换选择孩子；* 登录后隐藏侧边栏操作菜单，直接进入「我的孩子」看板；只读、卡片式布局。
   - 课务与进度：请求 `students` 表，以进度条或环形图展示 `total_hours`、`used_hours`、`remaining_hours`；展示最新教案执行进度与历史下载列表（PDF/预览）。
3. **历史记录与软删除闭环**
   - **前端**：教师端「历史教案」列表每一条提供「作废」按钮；管理员可对已作废教案执行物理删除。
   - **后端**：作废接口对 `assessments` 执行 **UPDATE**，将 `status` 从 `active` 改为 `void`；管理员可对已作废（`status='void'`）教案执行物理删除（DELETE）。
   - **数据隔离**：所有拉取历史数据的查询必须带 **`WHERE status = 'active'`**；作废记录仅保留备查，不参与雷达图与折线图计算；管理员可传 `status` 参数查看已作废记录。

**阶段交付物**：学生数据看板组件（可交互雷达图 + 时间轴折线图）；家长专属 Web 端（图表 + 课时进度）；完善的历史档案库（按时间筛选 + 作废/归档闭环）。

### 4.4 第五阶段：联调测试与交付（拆解实施路径）

**业务目标**：从开发环境迈向生产环境。通过越权与压力测试排查隐患，完成全局优化，在 Zspace NAS 上生产级部署，达到高可用、高安全后正式移交。

#### 4.4.1 模块一：四级权限安全与越权测试（保障数据护城河）

**目标**：验证前端路由与后端接口均按角色隔离，无垂直/水平越权。

**步骤一：垂直越权检查清单**

| 序号 | 测试场景 | 操作方式 | 预期结果 | 验收 |
|------|----------|----------|----------|------|
| V1 | 教师访问管理员专属页面 | 教师登录后，浏览器地址栏输入「教案库管理」「账号管理」等管理员路由 | 前端路由守卫拦截，跳转 403 或无权限页 | □ |
| V2 | 教师调用管理员专属接口 | 教师 Token 下请求 `POST /api/users/create`、`PUT /api/homepage/update`、`POST /api/teaching-targets/import` 等 | 后端返回 **403**，不执行操作 | □ |

## 5. 接口规范

### 5.1 基础约定

- **Base URL**：*生产环境公网域名为 **lexingronghe.com**；API 建议统一使用 `/api` 前缀（如 `https://lexingronghe.com/api`）。*
- **认证方式**：**JWT (jsonwebtoken)**。登录成功后下发包含用户 `id` 与 `role` 的 Token，需登录的接口在请求头中携带该 Token。*密码存储采用 **bcrypt** 加密，见 §8。*
- **通用请求头**：需鉴权接口携带 `Authorization: Bearer <token>`（或项目约定的 header 名）。
- **通用响应格式**：建议统一 `{ code, message?, data? }`，具体在实现时约定。

### 5.2 接口列表（第二阶段权限与 CMS）

以下为第二阶段前后端约定核心接口，开发须严格按路径、方法及权限执行：

| 接口路径 | 请求方式 | 作用说明 | 权限要求 |
|----------|----------|----------|----------|
| `/api/auth/login` | POST | 验证账号密码，下发 JWT Token | 无（公开） |
| `/api/users/list` | GET | 获取系统所有账号列表 | 超管、管理员 |
| `/api/users/create` | POST | 创建新账号（教师、家长等） | 超管、管理员 |
| `/api/homepage/config` | GET | 官网拉取首页配置（hero、philosophy、facilities、why_choose_us、contact 等） | 无（公开） |
| `/api/homepage/update` | PUT | 更新官网首页轮播图或视频链接 | 管理员 |
| `/api/articles/publish` | POST | 发布富文本活动宣传页 | 管理员 |

第三、四阶段新增接口（模板上传、标签库、教案生成、图表数据、家长端课时与历史等）在开发时补充登记至本节。

### 5.3 评估与教案接口补充

| 接口路径 | 请求方式 | 作用说明 | 权限要求 |
|----------|----------|----------|----------|
| `/api/assessments` | GET | 历史教案列表；支持 `student_id`、`status`（`active`\|`void`\|`all`）筛选，仅管理员可传 `void`/`all`。响应增加 `total_count`、`latest_date` | 超管、管理员、教师（按 `teacher_id` 隔离）、家长（按 `parent_id` 隔离） |

## 6. 数据库环境与开发规范（MySQL 子项目）

本子项目为系统核心业务（四级权限、多维度标签检索、智能教案生成、家校数据追踪）提供底层数据存储，须保证强数据一致性、支持复杂关联查询（雷达图/折线图数据聚合），并严格落实软删除与数据隔离。

### 6.1 环境与规范

- **数据库版本**：MySQL 8.0+（推荐，支持更优的 JSON 处理与查询性能）。
- **字符集**：统一 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`（支持生僻字及特殊符号）。
- **存储引擎**：统一 **InnoDB**（必须支持事务与外键，保证教案生成时主表与明细表同时写入成功或同时回滚）。
- **命名规范**：表名、字段名全部小写字母 + 下划线（**snake_case**）。

### 6.2 核心表结构 (DDL)

执行顺序：先建 `users`、`teaching_targets`，再建依赖它们的外键表。

**（1）用户与权限表 `users`**

```sql
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '登录账号(手机号或工号)',
    password_hash VARCHAR(255) NOT NULL COMMENT '加密后的密码',
    role ENUM('super_admin', 'admin', 'teacher', 'parent') NOT NULL COMMENT '四级权限角色',
    status ENUM('active', 'frozen') DEFAULT 'active' COMMENT '账号状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '账号创建时间'
) COMMENT='系统用户与角色权限表';
```

**（2）学生档案与课时表 `students`**

```sql
CREATE TABLE students (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL COMMENT '学生姓名',
    parent_id INT COMMENT '关联家长账号ID',
    teacher_id INT COMMENT '关联主负责教师账号ID(用于数据隔离)',
    total_hours INT DEFAULT 0 COMMENT '总购买课时',
    used_hours INT DEFAULT 0 COMMENT '已消耗课时',
    remaining_hours INT DEFAULT 0 COMMENT '剩余可用课时',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES users(id) ON DELETE SET NULL,
    FOREIGN KEY (teacher_id) REFERENCES users(id) ON DELETE SET NULL
) COMMENT='学生基本信息与课务表';
```

**（3）四级教学标签与目标库 `teaching_targets`**

```sql
CREATE TABLE teaching_targets (
    id INT AUTO_INCREMENT PRIMARY KEY,
    major_category VARCHAR(50) NOT NULL COMMENT '一级：大类，取值见§5.4(7大类)：CVP/EL/FM/GM/VMI/AB/PSC',
    sub_category VARCHAR(50) NOT NULL COMMENT '二级：小类(如：平衡能力)',
    stage INT NOT NULL COMMENT '三级：阶段(如：1, 2, 3)',
    target_type ENUM('long_term', 'short_term') NOT NULL COMMENT '四级：长短期目标',
    target_number VARCHAR(50) DEFAULT NULL COMMENT '目标编号：长期编号或短期编号',
    content TEXT NOT NULL COMMENT '具体的训练目标内容',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) COMMENT='四级联动评估目标库';
```

**（4）评估与教案主表 `assessments`**

```sql
CREATE TABLE assessments (
    id INT AUTO_INCREMENT PRIMARY KEY,
    student_id INT NOT NULL COMMENT '被评估学生ID',
    teacher_id INT NOT NULL COMMENT '执行评估教师ID',
    assessment_date DATE NOT NULL COMMENT '评估日期(用于生成历史折线图时间轴)',
    status ENUM('active', 'void') DEFAULT 'active' COMMENT '状态：生效或作废(软删除机制)',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (student_id) REFERENCES students(id),
    FOREIGN KEY (teacher_id) REFERENCES users(id)
) COMMENT='历次教案与评估记录主表';
```

**（5）评估得分明细表 `assessment_details`**

```sql
CREATE TABLE assessment_details (
    id INT AUTO_INCREMENT PRIMARY KEY,
    assessment_id INT NOT NULL COMMENT '关联教案主表ID',
    target_id INT NOT NULL COMMENT '关联具体训练目标ID',
    score INT DEFAULT 0 COMMENT '该项得分(用于图表数据聚合计分)',
    FOREIGN KEY (assessment_id) REFERENCES assessments(id) ON DELETE CASCADE,
    FOREIGN KEY (target_id) REFERENCES teaching_targets(id)
) COMMENT='单次评估勾选的目标明细与得分';
```

**（6）活动宣传文章表 `articles`**（第二阶段 CMS 使用）

```sql
CREATE TABLE articles (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(200) NOT NULL COMMENT '文章/活动标题',
    content LONGTEXT NOT NULL COMMENT '富文本HTML内容',
    status ENUM('draft', 'published') DEFAULT 'draft' COMMENT '草稿/已发布',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) COMMENT='活动宣传页富文本内容表';
```

**（7）教案模板库表 `templates`**（第三阶段动态模板使用）

```sql
CREATE TABLE templates (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL COMMENT '模板名称',
    file_path VARCHAR(500) NOT NULL COMMENT 'NAS中模板文件绝对路径',
    file_type ENUM('excel', 'word') NOT NULL COMMENT '模板类型',
    scenario VARCHAR(100) COMMENT '适用场景(可选)',
    status ENUM('enabled', 'disabled') DEFAULT 'enabled' COMMENT '启用状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) COMMENT='教案Excel/Word模板库';
```
*模板文件存储根目录由环境变量配置，已确认 NAS 路径：`/SATA存储11/web/teaching/templates`。*

```sql
CREATE TABLE homepage_config (
    id INT AUTO_INCREMENT PRIMARY KEY,
    config_key VARCHAR(50) NOT NULL UNIQUE COMMENT '配置键(如: banner_1_url, video_main_url)',
    config_value VARCHAR(500) NOT NULL COMMENT '配置值(图片或视频URL)',
    remark VARCHAR(100) COMMENT '备注',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
```

- **前端**：*官网已为 Vue 架构*；管理端/家校端统一使用 Vue.js + ECharts，不得引入文档未指定的前端框架。
- **禁止事项**：不得绕过 RBAC 做越权访问；不得在未约定处新增接口或字段；教师仅可作废教案，管理员可对已作废教案物理删除；图表与历史列表仅统计 `status = 'active'` 的评估；教师端查询学生/教案必须带 `teacher_id = 当前用户 id`，禁止水平越权。*用户密码须使用 **bcrypt** 加密后存入 `users.password_hash`。*
- **与现有系统对接**：首页素材替换需与现有 Vue 官网首页结构一致（轮播图/视频占位由前端从 `homepage_config` 或 `/api/homepage/config` 拉取渲染）。*首页动态化实施须遵循「不改变设计布局」约束，详见 §3.1 及 `docs/首页动态化项目规划.md`。*
