# 云服务器上 SSL 证书路径怎么填

Nginx 里需要两行：

- **`ssl_certificate`**：填 **`fullchain.pem`**（完整证书链，浏览器信任用这份）。
- **`ssl_certificate_key`**：填 **`privkey.pem`**（私钥，权限要严，勿泄露）。

下面按**常见申请方式**说明去哪里找。

---

## 一、用 Certbot（Let’s Encrypt）在 Linux 上申请的

这是最常见情况，证书一般在：

```text
/etc/letsencrypt/live/<证书目录名>/fullchain.pem
/etc/letsencrypt/live/<证书目录名>/privkey.pem
```

**`<证书目录名>`** 往往是：

- 你执行 `certbot` 时 **`-d` 里写的第一个域名**，或  
- Certbot 提示里显示的 **Certificate Name**。

### 怎么确认（在云服务器 SSH 里执行）

```bash
# 列出 certbot 已管理的证书及路径
sudo certbot certificates
```

输出里会有 **Certificate Name** 和 **Domains**，下面会写 **Certificate Path** / **Private Key Path**，**直接复制到 Nginx 即可**。

或列出目录：

```bash
sudo ls -la /etc/letsencrypt/live/
```

进入对应目录应能看到 `fullchain.pem`、`privkey.pem`（有时是符号链接，**Nginx 里写这个路径即可**）。

### 填到 Nginx 示例

```nginx
ssl_certificate     /etc/letsencrypt/live/lexingronghe.com/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/lexingronghe.com/privkey.pem;
```

若 **`certbot certificates`** 里显示的名字是 `www.lexingronghe.com`，则目录可能是：

```text
/etc/letsencrypt/live/www.lexingronghe.com/
```

**以 `certbot certificates` 输出为准**，不要猜。

---

## 二、在阿里云 / 其它云控制台「下载证书」再上传到服务器

控制台会提供 **Nginx 版** 压缩包，解压后通常有：

- 类似 **`xxx.pem`** 或 **`xxx.crt`**（证书 + 链，或分开两个文件）
- **`xxx.key`**（私钥）

你需要：

1. 上传到服务器固定目录，例如：`/etc/nginx/ssl/lexingronghe.com/`  
2. 若厂商给「证书」和「链」两个文件，Nginx 常要**合并成一条链**再作为 `ssl_certificate`，或按厂商文档操作。  
3. 在 Nginx 里写**你实际放置的绝对路径**，例如：

```nginx
ssl_certificate     /etc/nginx/ssl/lexingronghe.com/fullchain.pem;
ssl_certificate_key /etc/nginx/ssl/lexingronghe.com/privkey.key;
```

（具体文件名以你下载的为准。）

---

## 三、检查 Nginx 能否读到

```bash
sudo nginx -t
```

若报 **找不到文件** 或 **permission denied**，检查：

- 路径是否拼错（区分大小写）；  
- 是否用了 `sudo` 运行 nginx；  
- 私钥权限是否过宽（一般仅 root 可读）。

---

## 四、和本仓库示例的关系

**仅 80 的配置**（`nginx.lexingronghe.com.conf.example`）**不需要**填证书路径。  
**带 443 的配置**（`nginx.lexingronghe.com.https.conf.example`）里写的是 **Certbot 常见路径**示例；把 **`/etc/letsencrypt/live/lexingronghe.com/`** 换成 **`certbot certificates` 里显示的真实目录名**，两处 `ssl_certificate` / `ssl_certificate_key` 保持一致即可。
