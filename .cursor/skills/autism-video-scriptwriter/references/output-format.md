# 剧本交付格式说明（人类编辑用）

本文件供**人类编辑**核对 AI 输出是否齐全，**不替代** [SKILL.md](../SKILL.md) 中的工作流铁律。模型以 SKILL 为准；此处强调**顺序、列名与可选块**。

---

## 1. 推荐整体顺序

1. **元信息**（五行走齐，便于归档）
2. **【版本 X】** 标题 + 预估行 + Hook 类型
3. **分镜主表**（列顺序见下）
4. **画面占比统计**（表 + 一句是否满足占比）
5. **即梦**（表内含 **或** 独立「按镜展开」二选一）
6. **推荐标签**（全片级）
7. **可选**：封面 9:16、Seedance 长提示、两版本差异说明

单版本交付时：只有 **【版本 A】**（或用户指定名），**不要**硬凑版本 B。

---

## 2. 主表列名（从左到右，勿改列名）

| 列名 | 含义 | 编辑注意 |
|------|------|----------|
| 画面 | 1、2、3… | 与总镜数一致 |
| 时间段 | `0″–6″` 这种 | 首尾闭合，总长约等于目标时长 |
| 口播文案 | 本镜旁白 | 对照 content-rules 禁用词 |
| 字幕文本 | ≤15 字/句为宜 | 口语压缩版，非逐字稿 |
| 画面描述 | 拍什么、信息图还是角色 | 标清**角色镜 / 信息镜**便于算占比 |
| [视觉]转场 | `[视觉:淡入淡出]` 等 | 与 rhythm-templates 枚举一致 |
| 即梦·图片 | 文生图整段 | **必须以** jimeng 统一风格前缀开头 |
| 即梦·视频 | 图生视频短句 | 每格 ≤2 句中文 |
| 推荐标签 | 本期话题词 | 可与首列备注「全片共用」 |

若表宽难排版：**主表可省略即梦两列**，改在第三节 **「即梦 · 按镜展开」** 按画面编号列出（内容须与分镜一一对应）。

---

## 3. 画面占比（必检）

- **角色画面**：需即梦「角色参考」的镜（出现妈妈、孩子等）。
- **信息画面**：无角色的信息图、大字报、纯 CTA 底等。

约束：**角色占比 &lt; 80%**，**信息占比 &gt; 20%**（相对总镜数）。

编辑可在表外加一列「类型」标注 `角色/信息`，方便自检。

---

## 4. 版本标题行模板（复制用）

```
【版本 B】
预估口播总字数：___ 字 | 预估时长：___ 秒
Hook 类型：误区纠正型（示例，按 hook-formulas 实际填写）
```

---

## 5. 可选块清单

| 块名 | 何时需要 |
|------|----------|
| 封面图提示词（9:16） | 要做抖音封面时 |
| 长视频/Seedance 提示 | 用户指定 Seedance、可灵等长提示时 |
| 两版本差异说明 | 一次交付含 A+B 时，3～5 行即可 |

---

## 6. 导入飞书 / Excel 的建议

- 从 Markdown **主表**复制时，**即梦·图片**格内常有逗号，建议用 **制表符 TSV** 导出或分表「文案表 / 即梦表」两 Sheet。
- **时间段**列用文本格式，避免被表格当成时间序列。
