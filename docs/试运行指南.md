# 试运行指南（本机）

按以下顺序操作即可在本机试运行**官网 + 管理后台 + 后端 API**。

---

## 一、环境准备

- **Node.js** 18+
- **pnpm**（`npm i -g pnpm`）
- **MySQL** 8.0+ 已安装并运行（本机或 192.168.5.2）

---

## 二、数据库初始化（只需做一次）

**方式一：一键初始化（推荐）**  
已为你准备好 `server/.env` 和初始化脚本，只需：

1. 编辑 **server/.env**，把 `MYSQL_PASSWORD=` 改成你的 MySQL 密码（其他项试运行可不动）。
2. 在项目根目录执行（需已安装 Node 依赖，见第四节）：
   ```bash
   cd server
   pnpm run init-trial-db
   cd ..
   ```
   脚本会自动：创建数据库（若不存在）→ 建表 → 建索引 → 插入超管账号（admin / Zzx1998.）。

**方式二：手动执行**  
若希望自己执行 SQL，可参考下方原步骤。

1. **创建数据库**（若尚未创建）  
   在 MySQL 中执行：
   ```sql
   CREATE DATABASE IF NOT EXISTS lexing_school DEFAULT CHARACTER SET utf8mb4;
   ```

2. **建表**（请将 `你的数据库名` 和连接参数按实际修改）  
   ```bash
   mysql -h 127.0.0.1 -P 3306 -u root -p 你的数据库名 < server/database/schema-mysql.sql
   ```

3. **建索引**（可选）  
   ```bash
   mysql -h 127.0.0.1 -P 3306 -u root -p 你的数据库名 < server/database/schema-mysql-indexes.sql
   ```

4. **插入超管**  
   ```bash
   cd server
   MYSQL_DATABASE=你的数据库名 MYSQL_PASSWORD=你的密码 JWT_SECRET=dev-secret pnpm run seed-mysql
   cd ..
   ```

---

## 三、配置后端环境变量

**已为你生成 server/.env**，默认包含本机试运行配置。你只需：

- 打开 **server/.env**，在 `MYSQL_PASSWORD=` 后填入你的 MySQL 密码（其他项可不改）。
- 试运行不要设 `NODE_ENV=production`，否则未设置或使用默认 JWT_SECRET 会无法启动。

---

## 四、安装依赖（只需做一次）

在项目根目录执行：

```bash
cd server        && pnpm install && cd ..
cd web/website   && pnpm install && cd ..
cd web/admin     && pnpm install && cd ..
```

---

## 五、启动三个服务

需要**三个终端**（或一个终端里开多个 tab），分别执行：

| 终端 | 命令 | 说明 |
|------|------|------|
| 1 | `cd server && pnpm dev` | 后端 API，默认 http://localhost:3001 |
| 2 | `cd web/admin && pnpm dev` | 管理后台，默认 http://localhost:5001 |
| 3 | `cd web/website && pnpm dev` | 官网，默认 http://localhost:5000（见 `web/website/vite.config.ts`） |

**Windows** 下若没有 bash，可直接用 PowerShell 或 CMD 执行上述命令。

- 管理后台已配置 Vite 代理：访问 5001 时，请求 `/api` 会转发到 3001，无需再设 `VITE_API_BASE_URL`。
- 官网端口以终端里打印的为准（仓库内 `vite.config.ts` 设为 5000）。

## 六、访问与登录

1. **官网**：浏览器打开终端里显示的地址（如 http://localhost:5000）。
2. **管理后台**：打开 http://localhost:5001 ，使用超管账号登录：
   - 用户名：`admin`
   - 密码：`Zzx1998.`

登录后可在后台使用账号管理、首页配置、文章管理等（权限见项目说明书 §2）。

## 七、常见问题

| 现象 | 处理 |
|------|------|
| 后端启动报「缺少 MySQL 配置」 | 检查 `server/.env` 中 `MYSQL_DATABASE` 等是否填写正确。 |
| 后端报「生产环境必须设置 JWT_SECRET」 | 试运行不要设 `NODE_ENV=production`，或确保 `.env` 里 `JWT_SECRET` 已设且不是默认值。 |
| 管理后台登录后请求 404 或跨域 | 确保先启动后端（3001），再访问 admin（5001）；admin 通过代理访问 API，不要直接改前端的 API 地址。 |
| 官网「LOGIN」点进去不对 | 官网登录按钮指向管理后台；试运行默认是 http://localhost:5001，若 admin 端口改了，在 `web/website` 下建 `.env.development`，写 `VITE_ADMIN_URL=http://localhost:5001`。 |
| 根目录 `start.sh` 报错或路径不存在 | `start.sh` 里写了 `/app/work/logs/...`，仅适用于特定环境；本机试运行请按「五」用三个终端分别启动。 |
| 模板管理「预览」提示暂无 PDF | 需安装 LibreOffice 以自动将 Word/Excel 转为 PDF，见下方「八、模板 PDF 预览（可选）」；或使用「下载原文件」获取模板。 |

## 八、模板 PDF 预览（可选）

若需在模板管理中**在线预览** Word/Excel 模板，需安装 LibreOffice，系统会在上传后自动生成 PDF 供预览。

**安装方式**：

- **Windows**：从 [LibreOffice 官网](https://www.libreoffice.org/) 下载安装，安装后 `soffice` 会加入 PATH；或设置环境变量 `LIBREOFFICE_PATH` 指向可执行文件路径（如 `C:\Program Files\LibreOffice\program\soffice.exe`）。
- **Linux**：`sudo apt-get install libreoffice`（Debian/Ubuntu）或 `sudo yum install libreoffice`（CentOS）。
- **macOS**：`brew install libreoffice` 或从官网安装。

**环境变量**（可选）：

- `ENABLE_PDF_CONVERSION=false`：禁用 PDF 自动转换。
- `LIBREOFFICE_PATH`：指定 LibreOffice 可执行文件路径（默认使用 `soffice` 或 `libreoffice`）。
