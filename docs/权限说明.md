# 权限说明

本文档描述「数字化教学管理与展示系统」的现行角色与权限设计，与项目说明书 §2 及前后端实现一致。

---

## 1. 角色定义

| 角色代码 | 角色名称 | 说明 |
|----------|----------|------|
| `super_admin` | 超级管理员 | 系统底层维护：存储/备份/系统参数、分配管理员账号；不直接操作业务数据。 |
| `admin` | 管理员 | 业务总控：教师/学生/家长账号、给教师和家长号创建连接、教案库、模板、首页配置、上传/审核活动宣传、生成教案。 |
| `teacher` | 教师 | 仅负责分配给自己的学生：评估、生成教案、查看学生图表、作废教案、上传活动宣传。 |
| `parent` | 家长 | 只读、仅自家孩子：查看成长图表、历史教案、课时进度。 |

---

## 2. 账号状态（users 表）

| 状态代码 | 含义 |
|----------|------|
| `active` | 正常，可登录与使用系统 |
| `frozen` | 冻结，不可登录 |

仅超级管理员、管理员可修改用户状态（在账号管理页编辑用户时选择「正常」或「冻结」）。

---

## 3. 前端（管理后台）菜单与路由权限

- **超级管理员**：仪表盘、账号管理、教案库管理、模板管理、学生管理、评估与教案、内容管理、首页配置。
- **管理员**：同上（无「系统底层」专属菜单时与超管可见菜单一致；若后续增加「数据库备份」等，仅超管可见）。
- **教师**：仪表盘、学生管理、评估与教案、内容管理。
- **教案库管理**（含一键清空）：仅超管、管理员；教师不访问。
- **家长**：仅「我的孩子」；登录后默认进入「我的孩子」看板，不显示仪表盘及其他业务菜单。

越权访问（如教师访问账号管理路由）时，前端跳转 **403 无权限** 页。

## 4. 后端接口权限矩阵

### 4.1 认证

| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/api/auth/login` | POST | 公开 | 登录，返回 JWT。 |
| `/api/auth/me` | GET | 已登录 | 获取当前用户信息。 |
| `/api/auth/password` | PUT | 已登录 | 当前用户修改自己的密码（需旧密码）。 |

### 4.2 用户/账号管理

| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/api/users/list`、`/api/users` | GET | 超管、管理员 | 用户列表，支持 role、status、search、page、limit。 |
| `/api/users/create` | POST | 超管、管理员 | 创建账号（用户名、密码、角色）。仅超管可创建管理员/超管。 |
| `/api/users/:id` | GET | 超管、管理员 | 获取指定用户信息。 |
| `/api/users/:id` | PUT | 超管、管理员 | 更新用户（仅 role、status）。 |
| `/api/users/:id/password` | PUT | 超管可改任意用户；其他角色仅可改自己（需旧密码） | 修改密码。 |
| `/api/users/:id` | DELETE | **仅超级管理员** | 删除用户；不可删除自己。 |

### 4.3 内容与首页

| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/api/articles`、`/api/articles/publish` 等 | GET/POST/PUT/DELETE | 超管、管理员、教师（按接口） | 文章/活动宣传。 |
| `/api/homepage/config` | GET | 公开 | 官网读取首页配置。 |
| `/api/homepage/update` 等 | PUT | 超管、管理员 | 更新首页轮播图/视频等。 |

### 4.4 业务数据（水平隔离）

- **教案库**：`/api/teaching-targets` 增删改查、导入、一键清空 — **仅超管、管理员**；教师不访问教案库管理。
- **模板**：`/api/templates` — 超管、管理员管理；教师可读列表。
- **学生**：`/api/students` — 教师仅能访问 `teacher_id = 当前用户 id` 的学生；家长仅能访问 `parent_id = 当前用户 id` 的学生。
- **教案**：`/api/assessments` 生成、列表、作废、删除 — 教师仅能操作自己负责学生的教案，仅可作废；家长仅可读列表（自己孩子的）。`DELETE /api/assessments/:id` 物理删除，**仅超管、管理员**，且**仅可删除已作废（status='void'）的记录**。
- **图表**：`/api/students/:id/charts/radar`、`/api/students/:id/charts/trend` — 同上，按学生归属校验教师/家长身份。

## 5. 特殊规则摘要

1. **创建管理员/超管**：仅超级管理员可创建角色为 `admin` 或 `super_admin` 的账号；管理员只能创建教师、家长。
2. **删除用户**：仅超级管理员可删除用户，且不能删除自己。
3. **重置他人密码**：仅超级管理员可为任意用户重置密码（无需旧密码）；其他角色仅能修改自己的密码且需提供旧密码。
4. **数据隔离**：教师、家长所有「按学生」的接口均在后端按 `teacher_id` / `parent_id` 校验，禁止跨人访问。
5. **教案删除**：仅超管、管理员可物理删除教案，且必须以作废为前提；教师仅可作废。
