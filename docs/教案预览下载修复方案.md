# 教案预览与下载修复方案（已实施）

> 本文档记录 IEP 教案预览、Excel 插入与下载的修复方案及实施结果。

---

## 代码审查摘要

| 文件 | 审查结论 |
|------|----------|
| `assessmentsController.js` | 领域归一化、IEP 固定布局、样式复制逻辑清晰；`getMasterCell` 已不再用于 IEP 写入（改为固定行列 + 主动 merge），可保留供其他场景使用 |

## 一、问题与根因

| 问题 | 根因 |
|------|------|
| 长期/短期目标无法插入 | 后端按短码（PSC）过滤，数据库存「个人自理(PSC)」等完整标签，导致 domains 为空 |
| 预览与下载文件名错误 | 未传 template_id 时取最新模板（可能为 Word）；CORS 未暴露 Content-Disposition |
| 下载文件名固定为「教案」 | 跨域时浏览器不暴露 Content-Disposition，前端回退为固定名 |

---

## 二、已实施修改

### 2.1 后端 `server/src/controllers/assessmentsController.js`

| 修改项 | 说明 |
|--------|------|
| **领域归一化** | `normalizeMajorCategory()` 兼容 `PSC` 与 `个人自理(PSC)`，构建 domains 时按归一化结果过滤 |
| **默认模板** | 未传 template_id 时仅取 `file_type='excel'` 的最近启用模板 |
| **IEP 固定布局** | 表头第 4 行，数据从第 6 行起；第 1/2/3 列分别为领域、长期目标、短期目标 |
| **短期目标分行** | 每个短期目标占一行；领域与长期目标列纵向合并 |
| **样式复制** | 以第 6 行第 1–7 列为参考，`copyCellStyle` + `applyRowStyle` 使插入行格式一致 |
| **下载命名** | `{学生姓名}-{老师姓名}-{日期}.xlsx`，buildAssessmentData 返回 teacherName |
| **调试日志** | 开发环境输出 template、targetRows、domains、excel 写入坐标 |

### 2.2 后端 `server/src/app.js`

| 修改项 | 说明 |
|--------|------|
| **CORS 暴露** | `exposedHeaders: ['Content-Disposition']`，使前端可读取下载文件名 |

### 2.3 前端 `web/admin/src/views/assessments/AssessmentGeneratePage.vue`

| 修改项 | 说明 |
|--------|------|
| **下载兜底** | Content-Disposition 解析失败时用 blob.type / 模板类型推断扩展名，兜底为 `{学生姓名}-{年月}.{ext}` |
| **预览** | Excel 时标题显示「教案预览 (Excel)」；空 HTML 时提示「模板已命中，但未渲染出可预览内容」 |
| **模板** | 仅展示 enabled 模板；默认选中首个 Excel 模板 |
| **关联长期目标** | 勾选长期目标时自动勾选关联短期目标；取消时同步取消关联短期目标 |

## 四、验证要点

1. 选择「个人自理(PSC)」下的长期/短期目标，预览与下载应正确显示目标内容
2. 下载文件名应为 `{学生姓名}-{老师姓名}-{年月}.xlsx`（需重启后端使 CORS 生效）
3. 若短期目标多于 7 行，自动插入行并合并领域与长期目标列
4. 勾选「关联长期目标」后，取消长期目标应同步取消关联短期目标
