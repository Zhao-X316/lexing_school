# 第五阶段压测步骤说明（§4.4.2）

**目标**：用接近真实的数据量与并发，验证模板渲染与 NAS I/O 稳定、无内存溢出与排版错乱。

---

## 一、准备条件

- 后端与 MySQL 已部署并可访问（本地或 NAS）。
- 至少 1 个教师账号、1 个学生（已关联该教师）、若干标签目标与 1 个已上传的教案模板。
- 可选：机构真实标签 Excel（按 §9 P3 表头）、含 `{#targets}`…`{/targets}` 的 Word 模板。

---

## 二、步骤一：海量标签与教案生成

### P1 导入真实标签库

| 项目 | 说明 |
|------|------|
| **操作** | 将机构 7 大类 + 大量长短期目标 Excel 按 §9 P3 表头（领域；项目；阶段；长期编号；长期方案；短期编号；短期方案）整理，在管理后台「教案库管理」中选择「Excel 导入」并上传。 |
| **观察** | 导入请求无超时；接口返回成功；数据库中 `teaching_targets` 表记录数与 Excel 行数一致，无字段截断或乱码。 |
| **验收** | □ 导入成功，无超时或字段截断 |

### P2 单次勾选大量目标生成教案

| 项目 | 说明 |
|------|------|
| **操作** | 教师端选择一名学生，在四级标签中勾选**数十至上百个**目标，点击「生成教案」并选择 Excel 或 Word 模板，完成导出下载。 |
| **观察** | 浏览器不卡死、不白屏；后端无内存溢出（可观察进程内存）；生成的 .xlsx/.docx 可正常打开，排版不错乱、目标列表完整。 |
| **验收** | □ 模板引擎无内存溢出；生成文件可正常打开，排版不错乱 |

### P3 Word 循环占位符

| 项目 | 说明 |
|------|------|
| **操作** | 使用含 `{#targets}`…`{/targets}` 的 Word 模板，勾选**多个目标**（建议 10～50 个）生成教案并下载 .docx。 |
| **观察** | 打开 docx 后，目标列表行数（或段落数）与勾选数量一致；格式（表格、字体等）保持与模板一致，无错位或重复。 |
| **验收** | □ 导出 docx 中目标列表行数与勾选数一致，格式保持 |

## 三、步骤二：并发与 I/O

### P4 多教师同时生成下载

| 项目 | 说明 |
|------|------|
| **操作** | 模拟 3～5 个教师同时请求「生成教案」并下载。可采用以下方式之一：<br>• **手工**：多台设备或多浏览器同时登录不同教师账号，同一时刻点击生成。<br>• **脚本**：用 k6、ab 或 curl 并发调用 `POST /api/assessments/generate`（需先取得各教师 Token 及 student_id、target_ids）。 |
| **观察** | NAS 磁盘 I/O、CPU 与网络带宽在可接受范围；所有请求均返回 200（或 201），无 5xx；响应时间在可接受范围内（如单次 &lt; 30s）。 |
| **验收** | □ NAS 可承受；响应时间可接受，无 5xx |

**简单并发示例（需先替换 TOKEN、STUDENT_ID、TARGET_IDS）**：  
适用于 **Linux / macOS** 或 **Git Bash**。Windows 下可直接用 PowerShell 多次调用接口，或用 k6、Postman 等做并发。

```bash
# 5 个并发请求（bash 下）
BASE_URL="http://localhost:3001"
TOKEN="教师A的JWT"
STUDENT_ID=1
TARGET_IDS="[1,2,3,4,5]"
for i in 1 2 3 4 5; do
  curl -s -X POST "$BASE_URL/api/assessments/generate" \
    -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
    -d "{\"student_id\":$STUDENT_ID,\"target_ids\":$TARGET_IDS}" &
done
wait
echo "Done"
```

### P5 图表接口并发

| 项目 | 说明 |
|------|------|
| **操作** | 多用户（或同一教师多次）同时请求**同一学生**的雷达图与折线图，例如：<br>• `GET /api/students/:id/charts/radar`<br>• `GET /api/students/:id/charts/trend` |
| **观察** | 所有请求返回 200；响应时间稳定（建议 &lt; 2s）；MySQL 慢查询日志无异常；应用无超时或 5xx。 |
| **验收** | □ 查询带索引，响应稳定，无慢查询导致超时 |

**说明**：索引已由 §4.4.3 D1 在 `schema-mysql-indexes.sql` 中定义；若尚未执行，请先执行索引脚本再压测。
